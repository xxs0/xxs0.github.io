addEventListener vs onclick -- 笔记

发表于 2016-05-08   |   分类于 Web前端

Event Listeners (addEventListener or IE’s attachEvent)

早期IE版本（小于9），使用attachEvent方法

ele.attachEvent('onclick', function() { /* do something * / });

大多数浏览器(包含IE9及以上)，可以使用addEventListener
ele.addEventListener(‘click’, function() { / do something / }, false);

使用DOM Level2事件，理论上能在任意单个元素上获取无限个事件。唯一的实际限制是客户端内存和其他性能考虑。每个浏览器都不一样
上面的例子可以用匿名函数展示。也可以用函数引用或者闭包监听事件。

var myFunctionReference = function() { /* do something * / };
ele.attachEvent('onclick', myFunctionReference);
ele.addEventListener('click', myFunctionReference, false);

addEventListener另一个重要特性是最后的参数，它控制对冒泡事件的反应。false对95%的情况有效。attachEvent没有相同的参数，或者当使用行内事件。

inline event

所有支持js的浏览器中，你都能在行内放置事件监听

<a id="testing" href="#" onclick="alert('do something inline');">Click me</a>

大多数有经验的开发者鄙视这种方法，但是它能把事做成，而且简单直接。这里不用闭包或匿名函数，你控制的范围被限制。

另一个方法等同行内js，除了你有更多控制范围，也能使用匿名函数，函数引用和闭包。

ele.onclick = function() { /* do something * / };

行内事件的重要缺点是不如事件监听，你只能操作分配的一个行内事件。行内时间被存储喂一个键值对，意味着它能被重写。

var ele = document.getElementById('testing');
ele.onclick = function () { alert('do something #1'); };
ele.onclick = function () { alert('do something #2'); };

当你点击元素，只能看到“do something ＃2”，重写了第一个onclick属性，也重写了原来的行内HTML属性。检查这里

那个最佳

这个问题涉及浏览器的兼容性和重要性，你现在或以后需要追加不止一个事件给一个元素？如果是，attachEvent和addEventListener是必要的，如果不是，行内事件也是一个妙招。

jQuery和其他js框架包含了不用浏览器执行DOM Level2 事件的方案，所有你能写出跨浏览器的代码不用担心IE的历史

$(ele).on('click', function() {/* do something * /});

不要为了一件事情动用一个框架！

function addEvent(ele, event, funct) {
  if (ele.attachEvent)
  return ele.attachEvent('on'+event, funct);
  else
  return ele.addEventListener(event, funct, false);  
}
